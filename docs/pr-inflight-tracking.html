7. Key Concepts at a Glance

NEW InflightEquity Event

New CQRS event type on InventorySnapshot. Stores per-symbol inflight quantities for both mints and redemptions. Deduped: no event if maps unchanged.

NEW set_inflight()

Directly replaces inflight balance at a venue without touching available. Used by snapshot polling (vs transfer() which moves between available/inflight).

NEW Staleness Guard

is_stale_for_symbol() rejects poll data where fetched_at < last_rebalancing. Prevents stale async polls from overwriting cleared inflight after a transfer completes.

NEW Stuck Redemptions Query

symbols_with_stuck_redemptions() scans DB for DetectionFailed/RedemptionRejected aggregates. Adds their quantities to inflight so the system doesn't re-trigger.

CHANGED MintRejected

Now: no balance change (shares never left Alpaca). Before: cancelled inflight back to available, which was incorrect since inflight was never started.

CHANGED RedemptionRejected

Now: keeps inflight (uncertain token location). Before: cancelled back to Raindex available. Operator must resolve manually; next snapshot corrects.

NEW TransferFailed handling

Tokens never reached Alpaca's wallet. Cancels inflight + sets last_rebalancing timestamp so staleness guard protects the restored balance.

CRITICAL Poll Ordering

Inflight polled FIRST, before balance snapshots. Balance snapshots trigger rebalancing checks — if they land before inflight is set, has_inflight()=false could trigger duplicates.
